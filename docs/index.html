Manual do TOPCLIMABC

Como o sistema funciona

🎯 O que é o TOPCLIMABC?

O TOPCLIMABC é um auditor independente de previsões meteorológicas. Todos os dias ele coleta o que cada modelo previu e compara com o que realmente aconteceu. Hierarquia de fontes reais: CEMADEN (pluviômetros físicos em solo) como fonte principal, com reanálise Open-Meteo Archive (ERA5-Land + INMET) como fallback quando o CEMADEN está indisponível. Sua validação manual tem prioridade absoluta sobre qualquer fonte automática.

1️⃣

Coleta de Previsões (🕗 08h todo dia)

O backend Python consulta Open-Meteo (ECMWF, GFS, ICON, Best Match) e OpenWeatherMap, armazenando o que cada modelo prevê para os próximos 16 dias, dividido por período do dia.

2️⃣

Coleta da Realidade (🌧️ Ontem)

Coleta os dados reais de precipitação de ontem, usando o Open-Meteo Historical (reanálise ERA5 — a mesma usada pela comunidade científica). Os dados são separados por período: Madrugada, Manhã, Tarde e Noite.

3️⃣

Auditoria e Score (🏆 Quem acertou?)

O sistema compara o previsto com o real e calcula um score de 0 a 100 para cada modelo, cruzando a concordância de classe de chuva (60%) com a proximidade de volume em mm (40%).

4️⃣

Publicação Automática (🌐 GitHub Pages)

Todo esse processo roda automaticamente via GitHub Actions e publica os resultados neste site. Nenhuma intervenção humana é necessária para atualizar os dados diariamente.

Estes são os modelos numéricos de previsão monitorados. São os mesmos dados que alimentam os apps de clima que você usa no celular.

🏆 Como o Score é calculado?

O score vai de 0 a 100 para cada período do dia (Madrugada, Manhã, Tarde, Noite). O score final do dia é a média dos 4 períodos.

📊 Acerto de Classe (60% do score)

Compara a categoria de intensidade prevista com a real:

☀️ Acerto exato de classe 100 pts
🌦️ Erro de 1 degrau (Seco↔Garoa) 50 pts
⛈️ Erro grave (Seco vs Forte) 0 pts

📏 Proximidade de Volume (40% do score)

Compara os milímetros previstos com os reais. A cada 1mm de diferença, perde 10 pontos de volume. Com 10mm de diferença, esse score vai a zero.

score_volume = max(0, 100 − (diferença_mm × 10))

Tabela de Classificação Final:

✅

85 — 100 pts: Excelente

Acertou a categoria e o volume muito bem. Modelo confiável para esse prazo.

🟢

70 — 84 pts: Bom

Acertou a essência mas errou um pouco o volume ou ficou 1 degrau acima/abaixo.

🟡

50 — 69 pts: Médio

Acertou a existência de chuva mas errou intensidade ou timing significativo.

🟠

30 — 49 pts: Ruim

Errou gravemente. Previu chuva e ficou seco, ou vice-versa com diferença grande de volume.

❌

0 — 29 pts: Péssimo

Falhou completamente. Previu sol e choveu muito, ou previu tempestade e ficou seco.

O dia é dividido em 4 períodos para análise mais detalhada. Isso permite ver se um modelo errou especificamente a tempestade da tarde, por exemplo.

🌙

Madrugada (00h — 05h59)

Chuvas noturnas e de madrugada. Geralmente ligadas a sistemas frontais e instabilidade de alta altitude. As menos impactantes no dia-a-dia.

🌅

Manhã (06h — 11h59)

Chuvas da manhã em BC/Itajaí são frequentes no verão (orografia da Serra do Mar). Impactam o deslocamento matinal.

☀️

Tarde (12h — 17h59)

Período mais crítico! No verão catarinense, as tempestades convectivas mais fortes acontecem entre 14h e 17h. Prever corretamente essa janela é o maior desafio dos modelos.

🌆

Noite (18h — 23h59)

Chuvas noturnas — sistemas que se intensificam ao anoitecer. Comuns em episódios de ZCAS (Zona de Convergência do Atlântico Sul).

❓ Por que os cards aparecem vazios na aba Hoje?
A aba Hoje mostra previsões do dia atual — que ainda não foram auditadas porque o dia não terminou. Os scores (%) ficarão em branco até amanhã, quando o sistema pode comparar com a realidade.

Os dados de chuva prevista (mm) por período já estão visíveis nos cards como indicação do que cada modelo espera para hoje.
❓ Por que o Ranking começa zerado?
O Ranking acumula dados ao longo do tempo. No primeiro dia de funcionamento, ainda não existem amostras suficientes para calcular médias. Aguarde alguns dias e o ranking será populado automaticamente pelo GitHub Actions.
❓ De onde vêm os dados reais?
Os dados de realidade vêm do Open-Meteo Historical — uma API que combina a reanálise ERA5 do ECMWF com dados observacionais. É o mesmo padrão usado pela comunidade científica global. Fica disponível 1-2 dias após o dia encerrar.
❓ O que é a "Linha da Persistência"?
A Persistência é uma previsão ingênua: "amanhã vai ser igual a hoje". Se um modelo meteorológico perder para a Persistência, significa que ele é inútil — você estaria melhor olhando pela janela do que usando o app.
❓ Com que frequência atualiza?
Uma vez por dia, automaticamente, via GitHub Actions. O pipeline roda às 8h (UTC-3) e segue 4 etapas: coleta de previsões → coleta da realidade → auditoria → publicação no site.
❓ Para que serve o botão "Registrar Clima"?
Permite que qualquer pessoa em BC ou Itajaí registre manualmente o clima do momento via Supabase. Esses registros ficam visíveis no Feed em Tempo Real e complementam os dados automáticos com observações humanas.
❓ O que são os 4 prazos: 1d, 3d, 7d, 15d?
Cada prazo representa quantos dias antes a previsão foi feita. Um score de 95% em 1 dia significa que o modelo acertou 95% quando previu 1 dia antes. O mesmo modelo pode ter 40% em 7 dias, mostrando que perde confiabilidade com o tempo.
